# 🤖 [AI 지시서] 카카오 주소검색(완전 무료) 기반 단지(Complex) 온보딩 및 변경 플로우 구현

## 1. 개요 및 목적
ParkingOwner 앱에서 사용자가 본인의 단지(아파트, 오피스텔 등)를 검색하고 연결하는 기능을 구현합니다.
- **최초 가입 시:** 소속된 단지가 없는 유저에게 강제로 노출되는 온보딩 플로우로 작동합니다.
- **가입 이후:** 햄버거 메뉴(Drawer)를 통해 언제든 '단지 변경 및 추가'를 할 수 있도록 지원합니다.
- 단지의 고유 식별 기준은 **'공식 주소(도로명 주소)'**이며, '현대아파트'와 같은 이름은 해당 주소의 **'별칭(Alias)'**으로 저장됩니다.

## 2. 기술 스택
- **Frontend:** Flutter, Riverpod (상태관리), GoRouter (라우팅), `webview_flutter` (또는 `flutter_inappwebview`)
- **Backend:** Node.js, Fastify, Prisma ORM, MariaDB
- **External API:** **카카오(다음) 우편번호 서비스 (완전 무료, API Key 불필요, 호출 제한 없음)**

---

## 3. 데이터베이스 스키마 요구사항 (Prisma)
유저가 여러 단지에 소속될 수 있으므로 현재 활성화된 단지를 추적해야 하며, 단지 테이블은 주소를 고유 식별자로 사용합니다.

```prisma
model User {
  id               String   @id @default(uuid())
  email            String   @unique
  // ... 기타 유저 정보
  currentComplexId String?  // 현재 활성화된(선택된) 단지 ID
  
  complexes        ComplexMember[]
}

model Complex {
  id            String   @id @default(uuid())
  roadAddress   String   @unique // 도로명 주소 (단지 구분의 절대적 기준)
  jibunAddress  String?  // 지번 주소 (보조)
  zipCode       String?  // 우편번호
  alias         String   // 단지 별칭 (예: "테헤란 현대아파트")
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  members       ComplexMember[]
}
```

## 4. 완전 무료 카카오 주소검색 연동 규격 (WebView 방식)
REST API는 일일 호출 제한 및 Key 발급이 필요하므로, **완전 무료인 '카카오(다음) 우편번호 서비스 JS 라이브러리'**를 Flutter WebView에 올려서 사용합니다.
- **라이브러리:** `https://t1.daumcdn.net/mapjsapi/bundle/postcode/prod/postcode.v2.js`
- **구현 방식:** 로컬 HTML을 로드하여 주소 선택 시 `JavascriptChannel`을 통해 Flutter로 데이터(도로명, 지번, 우편번호)를 전달(PostMessage)합니다.

## 5. 구현 플로우 및 요구사항

### Step 1. 진입 라우팅 로직 (GoRouter)
- **최초 로그인 (온보딩):** 로그인 성공 후 `User` 정보 조회 시 `currentComplexId`가 `null`이거나 소속된 단지가 없다면, 홈 화면 진입을 막고 **단지 검색 화면으로 강제 리다이렉트(Redirect)** 합니다. (뒤로가기 불가)
- **햄버거 메뉴 (단지 변경):** 홈 화면 우측 상단 햄버거 메뉴(Drawer)에 `[🏢 단지 변경 / 추가]` 버튼을 배치합니다. 클릭 시 동일한 단지 검색 화면으로 이동합니다. (이때는 뒤로가기 가능)

### Step 2. 주소 검색 (Flutter WebView)
- "카카오 주소검색으로 찾기" 버튼 클릭 시 WebView 모달을 띄우고, JS 브릿지를 통해 주소 선택 완료 이벤트를 수신합니다.

### Step 3. 주소 등록 여부 확인 (Backend API)
- 전달받은 `roadAddress`를 백엔드로 보내 DB에 이미 등록된 주소인지 확인합니다.
- **API 명세:** `GET /api/v1/complexes/check?roadAddress={encoded_address}`

### Step 4-A. 미등록 주소일 경우 (새 단지 등록)
- "미등록 주소" 뱃지 노출 및 **[단지 별칭 (필수)]** 텍스트 입력 폼 제공.
- 폼 제출 시 새 `Complex` 생성 + 유저를 `단지관리자`로 등록 + 유저의 `currentComplexId`를 새 단지로 업데이트.
- **API 명세:** `POST /api/v1/complexes`

### Step 4-B. 기등록 주소일 경우 (기존 단지 가입)
- "파킹오너 등록됨" 뱃지 및 DB에 저장된 `alias`(별칭) 노출.
- `[입주민으로 가입 요청하기]` 또는 `[초대코드 입력]` 액션 제공.
- 완료 시 유저의 `currentComplexId`를 해당 단지로 업데이트.
