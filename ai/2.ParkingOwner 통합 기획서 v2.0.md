## 3. 사용자 역할 체계

### 3-1. 역할 정의

| 역할 | 수 | 주요 권한 |
|------|-----|-----------|
| **앱관리자** | 개발자 1명 | 시스템 전체 관리, 이의신청 처리 |
| **단지관리자** | 1명 / 단지 | 단지 생성, 관리인 임명, 모든 기능 접근 |
| **주차관리인** | N명 / 단지 | 차량 전체 관리, 세대원 승인, 실시간 스캔 |
| **세대원** | N명 / 단지 | 차량 조회/스캔(조회 전용), 본인 세대 차량 등록 요청 |

### 3-2. 역할 관계

```
앱관리자 (개발자)
    └── 단지관리자 (1명/단지)
            ├── 주차관리인 (N명/단지)  ← 단지관리자 겸직 가능
            └── 세대원 (N명/단지)      ← 주차관리인 겸직 가능
```

> - 단지관리자는 주차관리인 역할 겸직 가능
> - 주차관리인은 세대원이거나 외부 고용인 가능

## 9. 화면 구성 (관리자 앱 — 핵심 화면)

앱의 주요 화면은 **하단 5개의 탭(Tab)**과 **우측 상단 햄버거 메뉴(Drawer)**를 중심으로 구성됩니다.

### 📱 Screen 01 · 로그인
**목적:** 관리자 인증 및 앱 진입
* **기능:** 이메일/ID 로그인, 카카오/애플 소셜 로그인 (비회원 클릭 시 자동 가입 연동)
* **UI:** 딥 블루 그라디언트 배경(Light) / 단색 배경(Dark)의 글래스모피즘 카드 폼

---

### 📱 Screen 02 · 🏠 홈 (대시보드 탭)
**목적:** 주차장 현황을 한눈에 파악하고, 승인 대기 업무로 빠르게 진입
* **실시간 통계 카드:** 등록차량 수 / 방문차량 수 / 미등록 감지 수 / 전체 슬롯 사용률
* **긴급 알림 배너:** 방금 감지된 미등록 차량 등 즉각적인 조치가 필요한 항목 노출
* **승인 대기 요청 (바로가기):** `차량 등록 요청(n건)`, `세대원 가입 요청(n건)` 카드를 배치하여 클릭 시 각각 [차량관리], [단지관리] 탭으로 즉시 이동
* **최근 입출차 피드:** 최신 차량 출입 기록 리스트 (시간순)

---

### 📱 Screen 03 · 📷 스캔 (현장 업무 탭)
**목적:** 카메라로 번호판을 실시간 인식하고 현장에서 즉시 조치
* **실시간 스캔:** 카메라 뷰파인더 + OCR 인식값 및 신뢰도 표시 + 수동 수정 지원
* **차량 상태별 UI:** 등록(파랑), 방문(노랑), 미등록(빨강)으로 스캔 프레임 색상 자동 변경
* **현장 빠른 액션 (Quick Actions):**
  * `[방문차량 빠른 등록]`: 미등록 차량 현장 임시 등록 및 방문증 발급
  * `[무단주차 처리]`: 경고 스티커 발부 기록 및 시스템 알림 전송
* **차량 정보 및 히스토리:** 인식된 차량의 소유자 정보, 최근 입출차 이력, 미등록 감지 이력 제공

---

### 📱 Screen 04 · 🚗 차량관리 탭
**목적:** 단지 내 출입하는 모든 차량의 데이터 및 등록 요청 관리
* **차량 등록 요청 처리:** 입주민이 신청한 차량/방문차량 등록 요청 승인 및 반려
* **차량 리스트 관리:** 세대원 차량 및 방문 차량 리스트 (등록/수정/삭제)
* **연락처 연동:** 리스트에서 차주 연락처 클릭 시 즉시 전화/문자 발송 기능 지원
* **뱃지 연동:** 미처리된 차량 관련 요청이 있을 경우 하단 탭바에 숫자 뱃지 표시

---

### 📱 Screen 05 · 🏢 단지관리 탭
**목적:** 사람(세대원)과 공간(주차구역)에 대한 통합 관리
* **세대원 가입 요청 처리:** 앱 가입을 요청한 세대원의 정보(동호수 등) 확인 후 승인/반려
* **세대원 관리:** 단지 내 세대원 리스트 조회 및 권한 관리
* **주차 구역 및 슬롯 설정:** 
  * 주차 구역 그룹화 및 이름 지정 (예: 지상, 지하 B1, 지하 B2 / 초기 기본값 1개)
  * 구역별 주차 가능 대수(슬롯) 설정 및 현황 관리
* **뱃지 연동:** 미처리된 세대원 가입 요청이 있을 경우 하단 탭바에 숫자 뱃지 표시

---

### 📱 Screen 06 · 💬 소통 탭 (채팅 및 알림)
**목적:** 단지 내 그룹 소통 및 시스템 알림 수신 (상단 서브 탭으로 분리)

**A. [채팅·문의] 서브 탭**
* **채널 분리 운영:** 단체방 피로도를 줄이기 위해 목적별 채널 운영
  * `📢 공지사항`: 관리소 → 전체 세대원 (읽기 전용)
  * `👮 업무 소통방`: 관리소 ↔ 현장 근무자 (업무 일지, 스티커 발부 보고 등)
  * `💬 1:1 문의`: 관리소 ↔ 개별 세대원 (프라이빗 민원 처리)
* **FAB (우측 하단):** `[➕]` 버튼을 눌러 새로운 공지사항 작성 또는 1:1 채팅 시작

**B. [알림 내역] 서브 탭**
* **알림 리스트:** 세대원 가입, 차량 등록/삭제, 경고장 발부 등 시스템 알림
* **카테고리화:** `[시스템]`, `[차량]`, `[경고]` 등 말머리와 아이콘 색상으로 시각적 구분
* **FAB (우측 하단):** `[⚙️]` 버튼을 눌러 알림 설정 화면으로 이동 (단지별/개인별로 원하는 알림만 On/Off)

---

### 📱 Screen 07 · 햄버거 메뉴 (Drawer)
**목적:** 탭바에 포함되지 않은 부가 기능 및 앱 설정 접근
* **위치:** 모든 화면의 우측 상단 `[≡]` 버튼 클릭 시 우측에서 슬라이드 인
* **구성:**
  * **단지 관리:** 🏢 단지 정보 / 👥 세대원 관리(바로가기) / 📊 리포트(통계)
  * **앱 설정:** ⚙️ 설정(테마/언어) / 📖 이용 가이드 / 💬 고객센터
  * **푸터:** 🚪 로그아웃

## 10. 내비게이션 구조 (UI/UX 공통 정책)

### 10-1. 하단 탭 바 (Bottom Navigation)
업무 목적에 따라 명확히 분리되며, 확인이 필요한 업무는 **숫자 뱃지**로 표시됩니다.

| 탭 | 아이콘 | 주요 역할 | 뱃지 표시 기준 |
|----|--------|-----------|----------------|
| **홈** | 🏠 | 대시보드 및 실시간 요약 | — |
| **스캔** | 📷 | 차량 번호판 스캔 및 즉각 조치 | — |
| **차량관리**| 🚗 | 차량 리스트 및 등록 요청 처리 | ✅ 미처리 차량 등록 요청 건수 |
| **단지관리**| 🏢 | 세대원 리스트, 주차구역 설정 및 가입 요청 처리 | ✅ 미처리 세대원 가입 요청 건수 |
| **소통** | 💬 | 단지 내 채팅 및 시스템 알림 | ✅ 안 읽은 메시지 및 알림 건수 |

### 10-2. 헤더 및 FAB (Floating Action Button) 정책
* **헤더 일관성:** 모든 화면의 우측 상단에는 `[햄버거 메뉴(≡)]` 버튼을 고정 배치하여 학습 비용을 낮춥니다.
* **FAB 활용:** 화면 우측 하단에 떠 있는 동그란 버튼(FAB)을 사용하여, 스크롤 위치와 무관하게 해당 탭의 가장 중요한 액션(새 채팅 작성 `➕`, 알림 설정 `⚙️` 등)을 쉽게 실행하도록 유도합니다.

### 10-3. 화면 전환 흐름
```
로그인
  └─▶ 🏠 홈 (대시보드)
        ├─▶ 📷 스캔 탭 → [입차/출차/임시등록/무단주차 마킹]
        ├─▶ 🚗 차량관리 탭 → [차량 등록 승인] / [차량 리스트 및 연락처 관리]
        ├─▶ 🏢 단지관리 탭 → [세대원 가입 승인] / [주차구역 및 슬롯 설정]
        ├─▶ 💬 소통 탭
        │     ├─▶ 채팅·문의 서브 탭 → [공지/업무방/1:1채팅] (FAB: 새 채팅)
        │     └─▶ 알림 내역 서브 탭 → [시스템/차량/경고 알림] (FAB: 알림 설정)
        └─▶ ≡ 햄버거 메뉴 (모든 탭에서 접근 가능) → [단지정보/리포트/설정/고객센터/로그아웃]
```
